# Assistant

Локальный ассистент для macOS (Apple Silicon). Menu bar + прозрачный overlay,
захват микрофона, локальный ASR (за протоколом), OCR по хоткею, стриминг ответа LLM.

## Требования

- macOS 14.2+ (Apple Silicon)
- Swift 6.x / Xcode 26
- Для локального LLM: запущенный Ollama или llama.cpp server на `http://127.0.0.1:11434/v1`


```sh
swift build
swift test
swift run Assistant
```

## MVP-проверка

1. Поднимите LLM. Вариант А — локальная llama:

   ```sh
   ollama serve            # слушает :11434
   ollama pull llama3.1
   ```

   Вариант Б — ChatGPT: в overlay нажмите шестерёнку -> `Открыть страницу ключей
   OpenAI`, создайте API-ключ, вставьте его и нажмите `Сохранить`.

2. Запустите: `./scripts/run.sh`
3. Индикатор справа в overlay показывает состояние LLM: `жду llama на :11434`,
   `нужен ключ OpenAI` или `LLM готов`. Пока не `готов` — запрос не уходит.
4. Кнопки overlay:
   - `Слушать` / `Стоп` — микрофон
   - `Экран` — снять область вокруг курсора, распознать и спросить LLM
   - шестерёнка — подключение провайдера (порт llama / ключ OpenAI)
   - `x` — скрыть overlay

Приложение живёт в menu bar как значок `AI`, без Dock-иконки. Выход: `AI` -> `Выход`.

Горячие клавиши: `⇧⌘A` — снять область и спросить LLM, `⇧⌘O` — показать/скрыть overlay.
Требуют Accessibility (`System Settings` -> `Privacy & Security` -> `Accessibility`).

## Провайдеры LLM

Приложение — оболочка, своего аккаунта нет. Два варианта, оба за одним протоколом:

- **Локальная llama** — OpenAI-совместимый сервер на выбранном порту (по умолчанию
  `11434`). Ключ не нужен, всё офлайн.
- **ChatGPT (OpenAI API)** — по вашему API-ключу платформы OpenAI. Ключ хранится
  только в Keychain, в код/логи/промпты не попадает.

Важно про ChatGPT: подключается именно **API-ключ платформы OpenAI**
(platform.openai.com, отдельный биллинг за токены). Вход в подписку ChatGPT
(chat.openai.com) API-доступа не даёт — это ограничение OpenAI, а не приложения.

## Что работает сейчас

- Menu bar приложение (accessory, без Dock), прозрачный overlay-`NSPanel`
- Микрофон → ресемпл 16 kHz mono → VAD → сегменты речи (тишина в ASR не идёт)
- OCR области экрана через Vision по хоткею, с debounce и кэшем по хешу кадра
- ContextManager (actor) с дедупликацией хвостов и обрезкой окна
- PromptBuilder: компактный промпт, схлопывание реплик, бюджет по длине
- LLM стриминг по SSE (OpenAI-совместимый), батчинг обновлений UI
- Keychain для ключа, логи без секретов


## Про скрытие overlay от записи экрана

`OverlayWindow` ставит `sharingType = .none` (включено по умолчанию). Это убирает окно
из большинства ШТАТНЫХ путей захвата — скриншот, screen recording, шаринг экрана через
системные API (Zoom/Meet/QuickTime). Но это НЕ гарантия невидимости:

- поведение менялось между версиями macOS — проверяйте на своей;
- аппаратный захват (грабер, камера на монитор) окно видит всегда;
- часть путей ScreenCaptureKit и сторонний софт могут окно захватывать.

То есть в типовом шаринге экрана окно скрыто, но обещать «100% не спалится» нельзя.

## Структура

```
Sources/Assistant/
  App/       — main, AppDelegate, AppCoordinator, DIContainer
  UI/        — Overlay (NSPanel + SwiftUI), MenuBar
  Audio/     — источники, ресемпл, VAD, пайплайн
  ASR/       — протокол, стаб, seam под whisper
  Vision/    — захват области, OCR, пайплайн
  LLM/       — протокол, OpenAI-совместимый клиент, SSE-парсер
  Context/   — окно контекста, менеджер (actor), сборка промпта
  Core/      — логи, Keychain, настройки, хоткеи
Tests/       — ContextWindow, PromptBuilder, SSEParser
```

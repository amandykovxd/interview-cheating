# Assistant

Локальный ассистент для macOS (Apple Silicon). Menu bar + прозрачный overlay,
захват микрофона, локальный ASR (за протоколом), OCR по хоткею, стриминг ответа LLM.

## Требования

- macOS 14.2+ (Apple Silicon)
- Swift 6.x / Xcode 26
- Для локального LLM: запущенный Ollama или llama.cpp server на `http://127.0.0.1:11434/v1`


```sh
swift build
swift test
swift run Assistant
```

## MVP-проверка

1. Поднимите LLM. Вариант А — локальная llama:

   ```sh
   ollama serve            # слушает :11434
   ollama pull llama3.1
   ```

   Вариант Б — ChatGPT: в overlay нажмите шестерёнку -> `Открыть страницу ключей
   OpenAI`, создайте API-ключ, вставьте его и нажмите `Сохранить`.

2. Запустите: `./scripts/run.sh`
3. Индикатор справа в overlay показывает состояние LLM: `жду llama на :11434`,
   `нужен ключ OpenAI` или `LLM готов`. Пока не `готов` — запрос не уходит.
4. Кнопки overlay:
   - `Слушать` / `Стоп` — микрофон; распознанный разговор виден в блоке «Разговор»
   - `Ответить` — ответ LLM по услышанному вопросу собеседника (без OCR)
   - `Экран` — снять область вокруг курсора, распознать и спросить LLM
   - `Очистить` (в блоке «Разговор») — сбросить накопленный контекст
   - поле снизу — свободный текстовый запрос к LLM, во время ответа кнопка меняется
     на «стоп» и прерывает генерацию
   - шестерёнка — подключение провайдера (порт llama / ключ OpenAI)
   - `x` — скрыть overlay

Типовой сценарий: включаешь `Слушать`, собеседник задаёт вопрос вслух, жмёшь
`Ответить` — LLM отвечает по транскрипту. Ответ стримится в основную область снизу.

Приложение живёт в menu bar как значок `AI`, без Dock-иконки. Выход: `AI` -> `Выход`.

Горячие клавиши: `⇧⌘A` — снять область и спросить LLM, `⇧⌘O` — показать/скрыть overlay.
Требуют Accessibility (`System Settings` -> `Privacy & Security` -> `Accessibility`).

## Провайдеры LLM

Приложение — оболочка, своего аккаунта нет. Два варианта, оба за одним протоколом:

- **Локальная llama** — OpenAI-совместимый сервер на выбранном порту (по умолчанию
  `11434`). Ключ не нужен, всё офлайн.
- **ChatGPT (OpenAI API)** — по вашему API-ключу платформы OpenAI. Ключ хранится
  только в Keychain, в код/логи/промпты не попадает.

Важно про ChatGPT: подключается именно **API-ключ платформы OpenAI**
(platform.openai.com, отдельный биллинг за токены). Вход в подписку ChatGPT
(chat.openai.com) API-доступа не даёт — это ограничение OpenAI, а не приложения.

## Что работает сейчас

- Menu bar приложение (accessory, без Dock), прозрачный overlay-`NSPanel`
- Микрофон → ресемпл 16 kHz mono → VAD → сегменты речи (тишина в ASR не идёт)
- OCR области экрана через Vision по хоткею, с debounce и кэшем по хешу кадра
- ContextManager (actor) с дедупликацией хвостов и обрезкой окна
- PromptBuilder: компактный промпт, схлопывание реплик, бюджет по длине
- LLM стриминг по SSE (OpenAI-совместимый), батчинг обновлений UI
- Локальный ASR на whisper (Metal + Accelerate), модель качается при первом запуске
- Текстовый чат с LLM в overlay, кнопка прерывания генерации
- Keychain для ключа, логи без секретов

## ASR (whisper)

Распознавание локальное. Ядро (`ggml` + `whisper`) собирается из исходников в
`Vendor/whisper` таргетом `WhisperCore` — без cmake и внешних SPM-пакетов.
Бэкенд: Metal на GPU + Accelerate.

Модель в бандл не кладём: при первом запуске качается в
`~/Library/Application Support/Assistant/Models` (`ggml-base.bin` по умолчанию).
Пока модель не готова, ASR работает на заглушке — приложение не падает, OCR и
текстовый чат доступны. Когда модель загрузилась, `ASREngineHolder` подменяет
движок на whisper на лету. Прогресс виден в overlay (`ASR: загрузка base 42%`).

Настройки в `WhisperASREngine.Config`:

- `language = "auto"` — язык намеренно не форсим. С жёстким `ru` английские
  термины начинают транслитерироваться в кириллицу.
- `initialPrompt` — подсказка декодеру со списком тех-терминов, поднимает
  точность на аббревиатурах и названиях.
- модели: `tiny` (75МБ), `base` (142МБ), `small` (466МБ), все multilingual.
  tiny/base быстрые, но хуже держат термины; small — лучше для реальной работы.

Проверено на M4: `Testing Whisper integration with Kubernetes and Docker
deployment` распознаётся на `tiny` дословно за ~0.2с. Интеграционный тест
`WhisperASREngineTests` гоняет настоящую модель на реальном аудио и
пропускается, если модель не скачана.

Про Metal: `ggml-metal.metal` компилируется в рантайме через
`newLibraryWithSource`, который не резолвит `#include` с диска, поэтому
`ggml-common.h` вшит прямо в шейдер. Ресурсный бандл `Assistant_WhisperCore.bundle`
обязан лежать внутри `.app` — это делает `scripts/run.sh`, иначе GPU не поднимется.

Чего пока нет: потокового режима с перекрытием окон (распознаём готовые сегменты
от VAD, partial-результатов нет), Core ML энкодера, system audio вторым источником.

## Про скрытие overlay от записи экрана

`OverlayWindow` ставит `sharingType = .none` (включено по умолчанию). Это убирает окно
из большинства ШТАТНЫХ путей захвата — скриншот, screen recording, шаринг экрана через
системные API (Zoom/Meet/QuickTime). Но это НЕ гарантия невидимости:

- поведение менялось между версиями macOS — проверяйте на своей;
- аппаратный захват (грабер, камера на монитор) окно видит всегда;
- часть путей ScreenCaptureKit и сторонний софт могут окно захватывать.

То есть в типовом шаринге экрана окно скрыто, но обещать «100% не спалится» нельзя.

## Структура

```
Sources/Assistant/
  App/       — main, AppDelegate, AppMenu, AppCoordinator, DIContainer
  UI/        — Overlay (NSPanel + SwiftUI), MenuBar
  Audio/     — источники, ресемпл, VAD, пайплайн
  ASR/       — протокол, whisper-движок, хранилище моделей, стаб-fallback
  Vision/    — захват области, OCR, пайплайн
  LLM/       — протокол, OpenAI-совместимый клиент, SSE-парсер, health-check
  Context/   — окно контекста, менеджер (actor), сборка промпта
  Core/      — логи, Keychain, настройки, хоткеи
Vendor/whisper — ядро whisper.cpp/ggml, собирается таргетом WhisperCore (MIT)
Tests/       — ContextWindow, PromptBuilder, SSEParser, WhisperASREngine
```
